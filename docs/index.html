Major MIDI

Major MIDI is a Daisy Patch SM firmware for playing Standard MIDI Files from SD card through a SoundFont 2 synth engine, with a compact front-panel UI, live MIDI input, loop playback, internal or external sync, and assignable CV/gate I/O.

This page is meant to be a direct user guide: what to put on the SD card, what the controls do, how the menus work, how sync works, and how to save your setup safely.

At a Glance

  • Plays .mid files from SD
  • Loads .sf2 SoundFonts from SD
  • 16-channel mixer with fast front-panel control
  • Per-channel program overrides
  • Loop playback with saved song metadata
  • UART MIDI in and external USB MIDI in
  • Internal or external sync
  • Phase 1 CV/gate routing

Quick Start

  1. Put at least one .mid file into 0:/midi.
  2. Put at least one .sf2 file into 0:/soundfonts.
  3. Insert the SD card and boot the module.
  4. Open the main menu with a long encoder press.
  5. Load a MIDI file.
  6. Load an SF2.
  7. Return to the performance screen.
  8. Press Play.
External sync note: if sync is set to external and no valid external clock is present, playback will not run. Either switch back to internal sync or provide MIDI clock / gate clock.

SD Card Layout

Major MIDI expects these folders on the SD card:

0:/midi
0:/soundfonts

Example:

0:/midi/825.mid
0:/midi/beat_loop.mid
0:/soundfonts/microgm.sf2
0:/soundfonts/another.sf2

The file browser ignores hidden AppleDouble files such as ._581.mid and ._microgm.sf2.

Boot Behavior

  • The OLED shows a splash screen.
  • The SD card is scanned for MIDI and SF2 files.
  • The current configuration is initialized.
  • Saved CV/gate config is read from 0:/major_midi_cv_gate.bin if present.
Important: CV/gate config is not background-saved during playback. Runtime SD writes were intentionally disabled because they collided with active SF2 and MIDI streaming. Use Save All.

Performance Screen

The performance screen is the default operating view.

STP 120 M001 B1 V

Meaning:

  • STP / PLY: stopped or playing
  • 120: current BPM
  • M001: current measure
  • B1: current bank of 4 channels
  • V: current quick page

The next two lines show:

  • current MIDI file
  • current SF2 file

The table below shows the 4 visible channels as columns. Muted channels are displayed with a leading *, for example *01. Zero values are displayed as -.

Performance Workflow

Banks

  • B1 = channels 1-4
  • B2 = channels 5-8
  • B3 = channels 9-12
  • B4 = channels 13-16

Press a bank button once to select that bank.

Quick Pages

  • V volume
  • P pan
  • R reverb send
  • C chorus send
  • G program

Moving Between Pages

There are two ways to move between quick pages:

  • Encoder + B1
  • press the same bank button again quickly

Example:

  • press B1 once: bank 1, page stays as-is
  • press B1 again quickly: page advances
  • press B1 again quickly: page advances again

Knobs

K1..K4 always control the 4 visible channels.

  • On V: volume
  • On P: pan
  • On R: reverb send
  • On C: chorus send
  • On G: program override

Knobs use pickup / hysteresis behavior, so after a bank or page change they generally need to pass through the current stored value before they begin changing it.

Play and BPM

  • Play toggles transport play/stop
  • Encoder turn changes BPM in performance mode
  • BPM range is 20 to 300

Shift, Mute, and Loop

Shift Actions

  • Encoder + B1: cycle quick page
  • Encoder + B2: enter/exit mute mode
  • Encoder + B3: mute all or unmute all channels
  • Encoder + B4: enter loop edit

Mute All

Mute all is not a separate global audio mode. It directly toggles the per-channel mute state on all 16 channels.

Mute Mode

  • B1..B4: mute/unmute visible channels
  • Encoder + B1..B4: switch bank while staying in mute mode
  • Encoder press: return to performance

Mute mode does not overwrite the stored volume values.

Loop Edit

  • K1: loop start
  • K2: loop end
  • Encoder press: exit loop edit

Main Menu

Open the main menu with a long encoder press.

Main Menu Items

  • Load MIDI
  • Load SF2
  • FX Settings
  • Song Settings
  • SF2 Settings
  • CV/Gate
  • Save All

Menu Controls

  • Encoder turn: move cursor
  • Encoder press: select / confirm
  • Encoder turn while editing: change value
  • Play: back out of current page
  • long encoder press: leave menu entirely

Load MIDI / Load SF2

Choose a file with the encoder and press to load. A loading screen is shown. On success the UI returns to performance. On failure it remains in the menu and shows an error overlay.

FX Settings

  • Rev Time
  • Rev LPF
  • Rev HPF
  • Ch Depth
  • Ch Speed

Song Settings

Song Settings stores song-level metadata inside the current MIDI file using the custom MMID meta-event block.

  • BPM Ovr: per-song BPM override
  • Loop: enable / disable looping
  • Loop St: loop start measure
  • Loop End: loop end measure / loop length
  • Save To MIDI: write current song settings to the file

What Song Settings Persist

  • BPM override
  • loop enabled
  • loop start
  • loop beat/sub data
  • loop length
  • other supported Major MIDI metadata fields

SF2 Settings

This is the deeper synth and per-channel settings page.

  • Voices
  • Channel
  • Mute
  • Volume
  • Pan
  • RevSend
  • ChoSend
  • Program
  • Trans

Voices

Voices sets the maximum synth voice count, from 4 to 32. This is the main operator control for balancing polyphony and CPU load.

Program

Program can operate in two ways:

  • Program File: follow the MIDI file’s program changes
  • Program 000..127: force a program override

Program overrides are also available from the performance G page.

Transpose

Global transpose for melodic channels. Drum handling remains separate so channel 10 continues to behave as expected for kits.

CV/Gate

Phase 1 CV/gate routing is implemented. The page dynamically hides irrelevant options, so channel, CC, and sync resolution only appear when they are meaningful for the currently selected mode.

CV Inputs

  • CV In 1
  • CV In 2

Modes:

  • Off
  • MasterVol
  • BPM
  • Ch CC

Gate Outputs

  • Gate Out 1
  • Gate Out 2

Modes:

  • Off
  • Sync
  • Reset
  • Ch Gate

Sync output resolutions:

  • 1/4
  • 1/8
  • 1/16
  • 1/32
  • 1/64

CV Outputs

  • CV Out 1
  • CV Out 2

Modes:

  • Off
  • Pitch
  • CC

Pitch output supports:

  • selected channel
  • highest-note priority
  • lowest-note priority

Save All

Save All is available from the main menu and includes a confirm/cancel step.

It saves:

  • current MIDI song settings into the current MIDI file
  • current CV/gate config into 0:/major_midi_cv_gate.bin
Safety rule: playback must be stopped before saving. If playback is active, the UI will show Stop Playback First.

Sync

Major MIDI supports internal sync and external sync. The sync source switch is read from MCP23017 GPB1.

Internal Sync

Switch up.

  • transport uses internal BPM
  • encoder BPM changes apply directly

External Sync

Switch down.

  • MIDI clock is preferred if locked
  • otherwise gate clock on gate_in_1
  • no valid external clock means no free-running playback
Note: external sync behavior is much improved, but it remains one of the more complex parts of the firmware and is still an area worth continued testing.

MIDI Input

Supported live MIDI inputs:

  • UART MIDI on A2/A3
  • external USB MIDI on A8/A9

Live MIDI is serviced on the fixed control/audio path rather than a naive main-loop poll, which is why timing is now considerably tighter than earlier versions.

Handled message types include:

  • note on / note off
  • control change
  • program change
  • pitch bend
  • all notes off / all sound off
  • MIDI clock
  • MIDI start / continue / stop

Program Handling

Program state is preserved across mid-song starts, loop starts, and loop wraps.

Current program handling works in layers:

  1. file program changes from the MIDI file
  2. restored file program state on seek / loop start
  3. user program override on top

Performance Program Page

The G quick page is the fast live program page.

  • bank selects the 4 channels you are editing
  • knobs set program overrides for those channels
  • the bottom performance row shows program values while on G

Looping

Major MIDI no longer uses a naive stop-and-restart loop seam. Loop playback is pre-scheduled ahead of the boundary, and loop timing is based on the active BPM.

  • loop starts can begin immediately from the chosen position
  • next loop cycle is appended before the seam
  • still-active notes are terminated on wrap to prevent accumulation
  • program state is restored correctly across loop restarts

Typical Workflows

Play a Song

  1. Long-press encoder to open the menu.
  2. Load a MIDI file.
  3. Load an SF2.
  4. Return to performance.
  5. Press Play.

Mix the Visible Channels

  1. Select a bank with B1..B4.
  2. Cycle to V, P, R, or C.
  3. Use K1..K4.

Change Instruments for a Bank

  1. Select the target bank.
  2. Cycle to page G.
  3. Use K1..K4 to set the 4 visible programs.

Configure a Clock Output

  1. Open CV/Gate.
  2. Set a gate output to Sync.
  3. Choose the pulse resolution.

Save Your Setup

  1. Stop playback.
  2. Open the main menu.
  3. Select Save All.
  4. Confirm.

Troubleshooting

The song will not start in external sync mode

  • check whether MIDI clock is present
  • check whether gate clock is present
  • check whether the sync switch is set to external by accident

I hear crunching at high polyphony

  • reduce Voices in SF2 Settings
  • use a lighter SF2
  • test with a simpler MIDI file

The CV/Gate menu used to freeze the module

That was caused by background SD writes during active streaming. Runtime autosave was removed; use Save All instead.

Loop playback used to hang notes

Loop wrap now flushes still-active notes at the seam before continuing the next cycle.

External sync is still imperfect for my setup

This remains a good area for continued hardware testing. Verify the external source first with a known-good clock.
